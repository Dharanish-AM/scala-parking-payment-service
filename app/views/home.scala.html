Parking payment operations

A polished dashboard for the full parking payment lifecycle.

Create payments, calculate fees, process exits, inspect receipts, and handle refunds from one focused workflow. The interface is tuned for clarity, fast keyboard use, and production-style responsiveness.

Play Framework Slick ORM MySQL

Create payment

Start a new parking session with the vehicle entry timestamp.

POST /api/payments
◌ Create a payment to generate the record that the rest of the workflow uses.

Calculate fee

Use the payment id and the exit timestamp to calculate the total.

POST /api/payments/:id/calculate
◌ Fee calculation reads the stored entry time and fills in the latest payment card.

Other actions

Continue the same record through details, processing, receipt, and refund.
